How would someone submit transactions to L2?

  1. User signs a transaction using metamask
  2. User sends transaction to UD
  3. UD sends transaction to StarkEx
  4. StarkEx puts multiple user trasactions to a batch
  5. StarkEx generates cryptographic proof
  6. StarkEx submits the batch and the proof to L1

How would someone resolve a domain?

  1. Resolution Services syncs and aggregates data from multiple places
    • CNS
    • ZNS
    • New L1 Registry
    • L2 Registry
    • DAC Service
  2. Client makes a request to Resolution Service
  3. Resolution service resolves a domain and gives cryptographic proof that data is correct
  4. Client makes a request to ProxyReader contract to verify the data on L1
