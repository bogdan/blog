Implement it in one day

contract IRegistry is IERC721 {
  function set(key, value, tokenId);
  function get(key, tokenId) returns (string);
}

Resolution Algorithm

  • User puts example.crypto in a browser
  • Browser sends DNS request
    • type: A name: example.crypto
  • The gateway sends request on chain
    • Registry.get("dns.A", hash("example.crypto"))
    • Registry.get("dns.A.ttl", hash("example.crypto"))
  • Gateway sends response to the browser
    • value: 192.168.0.1, ttl: 128

Realistic Decentralization

  • Putting data on chain is expensive
  • Content storage motivation is not solved
  • Dynamic content is not even a thing
  • Decentralized technologies are slow

Content storage problem

What is the motivation of the network to store your content?

Ethereum has its own content storage motivation

Can this model be applied to content?

  • There is a way to motivate the network to store your content
  • There is a way to ask service provider for content storage confirmation
  • There is NO WAY to ensure service provider makes your content available

Dynamic Content

Every time you hear about some success in decentralized content storage and deliver no one is even telling you its only for static content

There is no technology to store dynamic content in a decentralized way in development
